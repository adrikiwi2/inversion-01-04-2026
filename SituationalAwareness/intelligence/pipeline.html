El pipeline en 4 pasos

F

Paso 1: Fetch — Recolección automática

Python scripts que llaman a APIs públicas gratuitas. Sin LLM, sin inteligencia. Solo datos crudos.

// fetch_daily.py hace 7 llamadas independientes: Yahoo Finance v8 × 103 tickers → precio, change_1d, change_1w Yahoo Finance v8 × 3 futuros → HG=F (cobre), GC=F (oro), NG=F (gas) CoinGecko API → BTC/USD, change_24h Blockchain.com API → hashrate (14d), EH/s CNN Fear & Greed API → score, rating SEC EDGAR full-text → filings SA LP + CORZ (hoy) Google News RSS × 6 queries → headlines por tema // Rate limiting: 1 seg cada 5 requests. Total: ~25 seg // Output: data/daily_2026-04-04.json (27,759 bytes)
Output Capa 1
27,759 bytes · 103 tickers · 18 noticias · 8 alertas
S

Paso 2: Estructuración — JSON normalizado

El fetcher ya estructura los datos en un schema predefinido. No hay "preprocesamiento" separado — la estructura ES el preprocesamiento.

{ "prices": { // 103 objetos {ticker, price, change_1d, change_1w} "LITE": { "price": 826.88, "change_1d": "+8.14%" }, "CORZ": { "price": 16.23, "change_1d": "+6.08%" }, // ... 101 más }, "commodities": { // cobre, oro, gas, uranio proxy }, "crypto": { "btc": {"btc_usd": 66952}, "hashrate": {"hashrate_ehs": 964.38} }, "macro": { "fear_greed": {"score": 19.3, "rating": "extreme fear"} }, "sec_filings": [], "news": { // 6 categorías × 3 headlines = 18 items }, "alerts": [ // 8 tickers con movimiento >5% ] }
Estructura
27,759 B → 22,614 chars de prompt (truncado news a 2/cat)
L

Paso 3: LLM — Análisis con GPT-5.4 via Codex CLI

El JSON completo se envía como contexto a GPT-5.4 junto con un prompt que define el schema de salida y el marco de análisis (la tesis de SA LP).

// El prompt tiene 3 componentes: SYSTEM_CONTEXT (fijo, 280 chars): "Eres un analista senior... TESIS: la infraestructura física es el cuello de botella... SA LP: 19 posiciones core..." DATOS (variable, ~22K chars): El JSON bundle completo del fetch, inline en el prompt SCHEMA (fijo, ~500 chars): "Escribe JSON con: market_summary, thesis_status, thesis_confidence, key_signals [{signal, direction, impact, tickers_affected, thesis_vector}], alerts_analysis, news_highlights, commodities_read, fear_greed_read, action_items, one_liner" FOCUS (fijo, ~120 chars): "Analiza movimientos >5%, noticias relevantes, commodities y Fear&Greed. Si F&G en extremos, comenta implicaciones." // Ejecución: codex exec --full-auto (stdin pipe) // Modelo: GPT-5.4 | Coste: $0 (ChatGPT Plus) // Tiempo: ~45 seg
Output LLM
22,614 chars prompt → 8,694 bytes análisis (compresión 2.6:1)
D

Paso 4: Dashboard — Renderizado visual

El HTML lee el JSON de análisis y lo presenta. Cada sección del dashboard mapea directamente a un campo del JSON.

Patrón emergente: La confianza baja con el horizonte temporal. Daily y weekly (datos duros de precio y hashrate) dan 8/10 porque los números son claros. Monthly baja a 7 porque los commodities se enfrían pero los lead times no normalizan (señal mixta). Quarterly baja a 6 porque falta el dato más crítico: capex de hyperscalers (Yahoo no lo sirve bien).

Implicación para trazabilidad: El gauge NO es un promedio. Cada frecuencia usa datos distintos, un prompt distinto con un FOCUS distinto, y el LLM evalúa la tesis desde un ángulo diferente. Un quarterly “stable 6” no invalida un daily “strengthening 8”.

Propuesta: Trazabilidad click-through

Hoy el pipeline es opaco: ves el output pero no puedes retroceder al input. El primer approach para hacerlo trazable:

Nivel 1: Source linking (bajo esfuerzo)

Cada signal/alerta del dashboard incluye un data-source que apunta al campo exacto del JSON bundle original. Al hacer click en “LITE +8.14%” se despliega el dato crudo de Yahoo Finance.

Nivel 2: Prompt transparency (medio esfuerzo)

Guardar el prompt completo enviado al LLM junto con el análisis (_meta.prompt_hash o prompt_daily_2026-04-04.txt). Permite re-ejecutar con el mismo input para verificar reproducibilidad.

Nivel 3: Reasoning chain (alto esfuerzo)

Pedir al LLM que incluya un campo reasoning por cada signal y veredicto. Ejemplo: “thesis_status = strengthening PORQUE 3 de los 5 mayores movers pertenecen a vectores distintos de la tesis (connectivity, data_centers, energy) y el F&G en extreme fear sugiere infravaloración.” Coste: +30% tokens de output.

Nivel 4: Diff-over-time (evolución)

Comparar automáticamente el análisis de hoy con el de ayer. “thesis_status pasó de stable a strengthening porque LITE entró en alerta y el F&G cayó de 25 a 19.” Requiere un segundo paso de LLM o lógica determinista simple sobre los JSON guardados.
